# FlashSell MVP 规划

## 目标
创建MVP（最小可行产品），验证核心产品-市场契合度

## MVP核心范围（3个月）

### 功能优先级

| 优先级 | 功能 | 描述 | 交付时间 |
|-------|------|-----|---------|
| P0 | 用户系统 | 注册/登录、支付宝订阅管理 | M1 |
| P0 | 选品搜索 | AI智谱API搜索、基础筛选（销量、价格、评分） | M1 |
| P0 | 产品详情 | 展示产品标题、图片、价格、BSR排名、评论数 | M1 |
| P0 | 收藏/看板 | 收藏产品、创建自定义看板 | M2 |
| P1 | 市场分析 | 品类销量分布、竞争强度评分 | M2 |
| P1 | 趋势追踪 | 品类趋势图、周环比/月环比 | M3 |
| P1 | 基础报告 | 选品报告导出（PDF） | M3 |
| P2 | Chrome插件 | 产品搜索、一键收藏 | M3（可选） |

### 技术栈选型

| 层级       | 技术选型                                        | 参考                                |
| -------- | ------------------------------------------- | --------------------------------- |
| **后端**   | Java + Spring Boot 2.7 + MyBatis + COLA 5.x | 基于 github.com/978655242/demo-cola |
| **前端**   | Vue 3 + Vite + Tailwind CSS                 | -                                 |
| **数据库**  | MySQL                                       | -                                 |
| **认证**   | JWT + Redis Session                         | -                                 |
| **AI搜索** | 智谱 API（ChatGLM）                             | 替代传统关键词搜索                         |
| **数据源**  | 智谱API + 电商平台公开数据                            | AI聚合多平台数据                         |
| **部署**   | Docker + 云服务器                               | -                                 |
| **支付**   | 支付宝（Alipay SDK）                             | 中国用户为主                            |

### 数据获取策略

| 数据源 | 用途 | 成本 |
|-------|------|------|
| 智谱 API | AI驱动的智能选品搜索、分析 | 按Token计费 |
| 电商公开数据 | 补充产品基础信息 | 免费 |
| 阿里云OSS | 图片、文件存储 | 按量计费 |

### MVP里程碑

| 周次 | 交付物 |
|-----|-------|
| W1-W2 | 项目初始化（Java后端+Vue前端）、数据库设计 |
| W3-W4 | 用户系统、支付宝订阅、智谱API集成 |
| W5-W6 | 选品搜索核心功能、产品详情页 |
| W7-W8 | 收藏看板、市场分析功能 |
| W9-W10 | 趋势追踪、报告导出 |
| W11-W12 | 测试修复、上线 |

### 后端架构（基于COLA）

```
demo-cola/
├── craftsman-adapter/       # Controller层，处理HTTP请求
├── craftsman-app/           # AppService，业务编排
├── craftsman-client/        # API接口定义（对外暴露）
├── craftsman-domain/        # 领域实体、规则
├── craftsman-infrastructure/# 技术实现（MyBatis、Redis、支付宝）
└── start/                   # 启动配置
```

### 前端架构（Vue 3）

```
flashsell-web/
├── src/
│   ├── views/              # 页面组件
│   ├── components/         # 通用组件
│   ├── stores/             # Pinia状态管理
│   ├── api/                # API调用
│   └── utils/              # 工具函数
├── Vite.config.ts
└── tailwind.config.js
```

### 关键指标（MVP验证）

| 指标 | 目标 | 验证方式 |
|-----|------|---------|
| 注册用户数 | 500 | 产品上线后1个月 |
| 活跃用户数 | 100 MAU | 产品上线后1个月 |

| AI搜索满意度 | >70% | 用户反馈收集 |
| NPS | >30 | 用户调研 |

## UI/UX 设计规划

### 设计目标
- 简洁高效，让卖家快速完成选品决策
- 数据可视化直观，减少认知负担
- 移动端适配，支持随时随地选品

### 设计工具
- Figma（推荐）或 Sketch

### 设计范围

| 页面 | 描述 | 优先级 |
|-----|------|-------|
| 首页/仪表盘 | 核心数据概览、快速搜索入口、推荐产品 | P0 |
| 选品搜索页 | AI搜索框、筛选条件、结果列表 | P0 |
| 产品详情页 | 产品信息、价格趋势、竞争分析、AI推荐 | P0 |
| 收藏看板页 | 我的收藏、看板管理 | P0 |
| 市场分析页 | 品类分析、趋势图表 | P1 |
| 订阅支付页 | 套餐选择、支付宝支付 | P0 |
| 个人中心 | 账户设置、订单记录 | P1 |
| 登录/注册页 | 手机号登录、支付宝授权 | P0 |

### 设计风格
- **主色调**：橙色/珊瑚色（活力、电商感）
- **字体**：思源黑体（中文）+ Inter（英文）
- **图标**：RemixIcon 或 Heroicons
- **数据图表**：ECharts 或 Chart.js

### 下一步

**当前阶段：UI/UX 设计**

1. 收集竞品截图（Keepa、Jungle Scout、卖家精灵）
2. 确定设计规范（颜色、字体、间距）
3. 绘制页面流程图（User Flow）
4. 设计线框图（Wireframe）
5. 设计高保真原型（High-fidelity Mockup）
6. 输出设计规范和组件库

1. 克隆 demo-cola 骨架项目
2. 初始化 Vue 3 前端项目
3. 配置 MySQL 数据库
4. 接入智谱 API
5. 开发选品搜索功能
6. 集成支付宝支付
